Ways to Create Phone Apps

When you think about having an app developed there are options about how the app can be developed that can be worth considering before you jump in.

Having created apps both ways listed below there are definitely benefits to each. For me personally my preference is web apps because I can more easily control the development and updating of the application - but in saying that there are many applications that are not suited to the web application approach.

Native Apps

Native apps are were you create the app using the devices native development language which differ on the iPhone, Android and other devices.

  • Use full complement of native functions such as camera, accelerometer, address book …
  • You can use storage on the phone for any photos/files.
  • The app will run faster.
  • App store approval process can hinder - while it helps monitor the quality it also can causes big headaches when rapidly developing new functionality.

Web Apps

With web technologies getting far richer there aren’t many things that you can’t do these days.

  • Create the app once for all platforms - you are targeting the phones browser so you can get greater coverage with one version of your app.
  • Quicker and easier to update - you can very quickly publish changes and fixes to your app as they aren’t held up by app stores.

Other options

There are always other options :) With phone apps you can create a web app and then using certain tools convert it to a native app. This can help with the issue of creating apps that work on multiple devices - which can be a blessing depending on the audience of your app.
